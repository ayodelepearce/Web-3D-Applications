3D Model Design Process

I created 3 original models using Blender 5.0 on my M1 Macbook Air. The models where: a retro game console, a pickaxe, and a sushi plate. I spent copius amounts of time researching model ideas on Youtube and then iterating them to make it more personal. I drew lots of inspiration from Polygon Runway on Youtube. I found it hard at times to work on my Mac as certain shortcuts in Blender behave differently or are missing.

For the retro console, I had to purge unused data in Blender to ensure that the image of Mario loaded in correctly. I also had objects within objects which where causing reflaction glitches and making the model look ugly. I fixed it by deling the inner geometry and seperating the top of the console from the bottom by using face select to make them seperate objects. Several objects where sharing the same material as I made them linked duplicates. I fixed it by making each of the duplicates their own seperate object and assigning each object with a specific material. The Mario image on the TV screen was glowing white as i Had set a value for emmission strength. I fixed it by setting emmission to 0 on the Principled BSDF shader. I also removed a duplicate Mario image that was plugged into the wrong socket.

Initially, I tried setting up my own custom nodes and textures for my pick axe material but then I noticed it was not compatible with GLTF export. So I rebuilt the textures of the pickaxe using samples from ambientCG. I was struggling to get the rice texture of my sushi model to export in GLTF, so I had to manually apply the displace and subdivision modifiers to ensure the displaced geometry was within the mesh. For the salmon color, I sourced a high quality picture of raw sushi and adapted it for my website. For the plate I applied Smart UV project to resolve the plate image texture stretching and warping. For all the models, I packed all the textures using File then External Data then Pack Resources. To guarantee they exported propleryl with the data embedded within the GLB files.

Design Rationale

The UI utilizes a Soft Minimalist aesthetic, I chose to use the fonts that I like to use when reading on my E reader to give it some personilisation. The background color of the Three.js viewer was changed from black to off-white (#f5f0eb) to improve the visibility of the 3D models and remain consistent with the rest of my page.

Typography:I used Papyrus for headings to make my headers look classy and AnticDidone for subheaders to make it more visually appealling. I used Fluid Typography (CSS Clamp) to ensure the site is fully responsive from mobile to ultra-wide displays. I also used courier font for the main text body as I like the simplicity and style of it.

Layout: I used Bootrap 5.3's fluid grid system with custom 25px border-radius on the cards to create a subtle depth effect. On larger screens, the 3d view and model description are side by sidle while on smaller screens they stack vertically.

Technical Decisions

MVC Architecture: My website uses a Model-View-Controller pattern, all model data is stored via an SQLite database and managed via PHP. The HTML/CSS is the frontend while Three.js is the controller that fetches data via AJAX dynammically updating the 3D model being seen.

PHP & SQLite Backend: A PHP abstraction later connects to the SQLite database to feed model data as JSON. data.pho supports single model queries via ID and dill model queries, making it a simple restful API.

Local Development: To ensure that everything was working as intended, I used MacOS's built in CLI "Terminal". I accessed and iterated on my website using Google Chrome via Terminal(php -S localhost:8000). This ensure that SQLite transactions and Three.js model loading was handled nicely withn a HTTP enviroment, bypassing the restrictions of trying to use Launch on Visual Studio Code.

Hardware-Aware Rendering: I used window.devicePixelRatio to ensure that the WebGL canvas matches the pixel density of high resolution displays. This ensures that the browser stops upscaling a lower resolution buffer, which was causing texture aliasing and blurry 3D models.

Texture Quality: I used anistropic texture filtering to prevent texture blurring at oblique angles. I applied sRGB color space correction for GLTF exports in Blender, this ensured that material colors rendered correctly in Three.js.

3D Viewer Features

Wireframe Toggle: Switches 3D model materials from standard sloid view to wirewarframe, allowing users to examine the underyling geometry of each of my models.

Lighting Toggle: Lighting is on by deafult when the button is pressed the ambient and directional lights are switched off to simulate a dark environment.

Rotate Button: Clicking the rotate button triggers a continuos rotation of the object on its Y Axis. This is done by using a boolean flag that is checked within the animation loop.

Reset View: Returns the camera to its original position, stops active rotations and returns the model back to its original orientation when clicked.

Orbit Controls: OrbitControls with damping via Three.js allow for smooth mouse-driven rotation, zoom and pan for full model inspection.

Animation (Pickaxe only): I added a short swing animation that mimics the motion of mining. The user can trigger it on demand by clicking the Play Animation button. It plays once per click, does not autoplay, and does not loop, matching the deliberate "one swing at a time" message in my model. In Blender I created an Empty PickaxePivot at the origin and parented the three pickaxe meshes to it so the whole tool rotates as one. I then keyframed the rotation, 0 degrees at rest, minus 50 degrees to lift back, plus 60 degrees on the strike, then back to 0. I exported with the model using GLTF's Sample Animations option so the rotation curves are baked into the GLB. I loaded the model when the pickaxe is the active model. I made the animation manual by using a dedicated button as autoplaying the swing on load would be distracting while the user reads the description.

Accessibility

Contrast: I used Dark-mode navbars and footers to provide contrast between text and background, to hel; with readability.

Responsive Design: I used Bootstrap 5.3's grid system to ensure that the website layout adapts correctly across different device and screen sizes. I used CSS clamp() to prevent text elements from becoming too small on mobile or too large on bigger displays.

Keyboard & Screen Reader Compatibility: Interactive elements such as navigation links and 3D control buttons are standard HTML elements that can be accessed via keyboard navigation. I applied ARIA labels to interactive 3D elements for screen reader compatibility.

Semantic HTML: My website uses semantic elements such as nav, main, section and footer to provide meaningful document structure that makes it easy to navigate.

Key Code Fixes and Deeper Understanding

Variable Scope Fix: Initially, modelCache and lightsOn were undeclared, causing a ReferenceError on every function call. ambientLight, mainLight and fillLight were declared as const inside initThreeJS(), making them inaccessible to toggleLights(). I moved all shared variables to module scope to resolve these errors.

Script Placement Fix: I placed the bootstrap script tag inside the footer nav element, which is invalid HTML. I moved it outside and after the footer to ensure correct parsing and execution order.

Blender Node Compatibility: I replaced procedural node setups including noise texture, color ramp and Bump nodes with simple Image Texture nodes connected directly to the Principled BSDF Base Color input. As the GLTF format only supports a limited set of Blender's node graph, so complex nodes are dropped when you try to export.

Model Orientation: I set default model rotation values to ensure each model faces the viewer correctly, this is due to the different coordinate systems between Blender (Z-up) and Three.js (Y-up).
